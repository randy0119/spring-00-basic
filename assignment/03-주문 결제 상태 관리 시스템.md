# 3. 주문 결제 상태 관리 시스템
## 문제 설명

> 주문은 생성 후 결제 대기 상태가 되며, 결제가 완료되면 결제 완료 상태가 된다. 이미 취소되거나 완료된 주문은 특정 전이만 허용된다.

## 요구사항
1. 주문 생성 API를 구현한다.
2. 주문 조회 API를 구현한다.
3. 주문 결제 API를 구현한다.
4. 주문 취소 API를 구현한다.
5. 주문 상태는 `CREATED`, `PAID`, `CANCELED` 중 하나다.
6. 이미 결제 완료된 주문은 다시 결제할 수 없다.
7. 취소된 주문은 결제할 수 없다.

## 확장 요구사항
1. 부분 취소 기능을 추가하라.
2. 결제 실패 이력을 저장하라.

## 테스트 포인트
1. 주문 생성 후 초기 상태 확인
2. 결제 성공
3. 중복 결제 실패
4. 취소 후 결제 실패

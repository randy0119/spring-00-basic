# 4. 송금 요청 시스템
## 문제 설명

> 사용자는 다른 사용자에게 송금을 요청할 수 있다. 요청받은 사용자는 이를 수락하거나 거절할 수 있다.

## 요구사항
송금 요청 생성 API를 구현한다.

송금 요청 목록 조회 API를 구현한다.

송금 요청 수락 API를 구현한다.

송금 요청 거절 API를 구현한다.

요청 상태는 `PENDING`, `ACCEPTED`, `REJECTED`, `EXPIRED` 중 하나다.

만료된 요청은 수락할 수 없다.

자기 자신에게 송금 요청은 생성할 수 없다.

## 확장 요구사항

요청 생성 후 10분이 지나면 만료 처리되도록 하라.

수락 시 송금 요청자 잔액을 검증하라.

## 테스트 포인트

정상 요청 생성

자기 자신 요청 실패

만료 후 수락 실패

중복 수락 방지

### 📁 `user` 패키지 개요

현재 위치인  
`spring_01_boot/src/main/java/com/example/spring_01_boot/user` 는  
스프링 부트 애플리케이션에서 **`user` 도메인**을 담당하는 패키지입니다.

여기에는 “사용자”와 관련된 **데이터·기능·API** 가 모여 있게 됩니다.

---

### 🔍 도메인(Domain)이란?

**도메인**이란, 비즈니스 로직에서 다루는 핵심 개념과 그에 대한 기능을 말합니다.

- 예시 도메인
  - `user`  : 사용자, 계정, 권한 등
  - `product` : 상품 정보, 재고 등
  - `order`   : 주문, 결제 흐름 등

스프링 부트 프로젝트에서는 **도메인별로 패키지를 분리**해서 관리하면:

- 코드 구조가 명확해지고  
- 변경 영향 범위를 파악하기 쉽고  
- 기능 추가·수정 시 유지보수가 편해집니다.

---

### 🧱 도메인 패키지 안에 들어가는 것들

일반적으로 하나의 도메인 패키지에는 다음과 같은 요소들이 함께 모여 있습니다.

- **Entity (엔티티)**  
  - 데이터베이스 테이블과 1:1(또는 유사)하게 매핑되는 클래스  
  - 예: `User`, `UserProfile` 등

- **Repository (리포지토리)**  
  - 엔티티를 **조회·저장·삭제** 하는 데이터 접근 계층  
  - 예: `UserRepository`

- **Service (서비스)**  
  - 도메인과 관련된 **비즈니스 로직**을 처리하는 계층  
  - 예: 회원 가입, 비밀번호 변경, 탈퇴 처리 등  
  - 예: `UserService`

- **Controller (컨트롤러)**  
  - HTTP 요청을 받아 Service 를 호출하고,  
    그 결과를 클라이언트에 응답하는 웹 계층  
  - 예: `UserController`

---

### ✅ `user` 패키지에는 어떤 것이 들어오나요?

`user` 도메인 패키지에는 보통 다음과 같은 파일들이 위치하게 됩니다.

- `User` 엔티티
- `UserRepository`
- `UserService`
- `UserController`
- (필요하다면) `UserDto`, `UserMapper`, `UserValidator` 등

---

### 정리

**도메인 패키지**는  
특정 비즈니스 영역(여기서는 “사용자”)과 관련된 **데이터와 기능을 한 곳에 모아 두는 구조**입니다.

이렇게 나누어 두면:

- “사용자 기능은 `user` 패키지에서 보면 되겠다.” 라는 **명확한 기준**이 생기고  
- 규모가 커져도 **코드를 찾기 쉽고, 유지보수가 쉬운 설계**가 됩니다.
